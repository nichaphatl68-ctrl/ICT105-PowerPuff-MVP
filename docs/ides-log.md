## Idea 2
Smart Cafeteria Queue System

### Problem Area
Long queues in the university cafeteria waste students’ time, especially during peak lunch hours.
Target Users

### Target Users
University students

### Current Alternative
Students physically wait in line to place orders

### Proposed IT Solution
A digital queue and pre-order system where students can: View cafeteria menus Place orders in advance Receive estimated preparation times Get notified when their order is ready

### Possible Technology
Frontend: HTML/CSS/JavaScript or Flutter Backend: Node.js or PHP Database: MySQL or Firebase QR code integration for order pickup

### Why This is Suitable
This idea is realistic for a semester project and demonstrates important IT skills such as real-time data handling, order management, notifications, and user-friendly interface design.

## Idea 3
Campus Navigation App

### Problem Area
New students and visitors often have difficulty finding classrooms, offices, buildings, and facilities on campus, leading to confusion and wasted time.

### Target Users
New university students Current students Visitors and parents

### Current Alternative
Students rely on campus maps, signboards, and asking others for directions.

### Proposed IT Solution
A mobile-friendly campus navigation app that allows users to: Search for buildings, classrooms, and facilities View an interactive campus map Get directions to destinations Locate important places such as cafeterias, libraries, and parking areas

### Possible Technology
Frontend: HTML, CSS, JavaScript, React Backend: Node.js or ASP.NET Core Database: MySQL or PostgreSQL Maps: Google Maps API or custom campus map integration GPS and location services

### Why This is Suitable
This project solves a common problem faced by students and visitors. It can be developed as a semester prototype with core features such as map display, location search, and navigation. The project demonstrates web/mobile development, database management, API integration, and user interface design.
